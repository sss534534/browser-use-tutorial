# browser-use 系统架构深度分析

## 📐 整体架构概览

browser-use 是一个模块化的 Python 框架，用于自动化浏览器任务。系统采用分层架构设计：

```

└───────────┼────────────────┼────────────────┼──────────────────┘
```

## 🏗️ 核心层级详解

### **第 1 层：应用入口**

| 组件 | 文件 | 职责 |
|------|------|------|
| **CLI (TUI)** | `cli.py` | 交互式终端用户界面，支持实时任务监控 |
| **MCP Server** | `mcp/server.py` | Model Context Protocol 服务器，支持与 Claude 集成 |
| **Sandbox** | `sandbox/` | 安全沙箱执行环境，支持 SSE 流式响应 |
| **InitCmd** | `init_cmd.py` | 项目初始化命令，从模板库获取示例 |

```python
# 使用示例
from browser_use import Agent, ChatBrowserUse

agent = Agent(
    task="Search for AI news",
    llm=ChatBrowserUse(),
)
await agent.run(max_steps=100)
```

### **第 2 层：核心业务逻辑（Agent Engine）**

#### **Agent Service** (`browser_use/agent/service.py`)

```

├── 初始化阶段
```

## 🔄 数据流和交互

### **完整执行流**

```
用户任务
    ↓
Agent.__init__()
    ├─ 初始化 BrowserSession
    ├─ 初始化 Tools
    ├─ 初始化 LLM
    ├─ 初始化 MessageManager
    └─ 初始化 EventBus
    ↓
agent.run(max_steps=N)
    ├─ 启动浏览器会话
    ├─ 执行��始动作
    ├─ 主循环 (每步)
    │  ├─ _prepare_context()
    │  │  ├─ 获取浏览器状态
    │  │  ├─ 提取 DOM
    │  │  ├─ 截图
    │  │  └─ 构建状态消息
    │  │
    │  ├─ _get_next_action()
    │  │  ├─ 构建 LLM 输入
    │  │  ├─ 调用 LLM
    │  │  ├─ 解析输出
    │  │  └─ 验证动作
    │  │
    │  ├─ _execute_actions()
    │  │  ├─ 执行每个动作
    │  │  ├─ 等待网络空闲
    │  │  └─ 收集结果
    │  │
    │  ├─ _post_process()
    │  │  ├─ 更新记忆
    │  │  ├─ 检查完成条件
    │  │  └─ 内存压缩（如需）
    │  │
    │  └─ _finalize()
    │     └─ 更新历史记录
    │
    ├─ 判断是否完成
    │  ├─ 如果完成: 运行 Judge，触发完成回调
    │  └─ 如果超过步数: 标记失败
    │
    └─ 返回 AgentHistoryList

AgentHistoryList
    ├─ final_result()        # 最终结果
    ├─ usage()              # Token 使用
    ├─ errors()             # 错误列表
    ├─ urls()               # 访问的 URLs
    └─ total_duration_seconds()
```

## 🎯 设计特点

### **1. 模块化和可扩展性**

- **解耦设计**：每个组件独立，通过清晰的接口通信
- **Lazy Loading**：使用 `__getattr__` 延迟导入重型模块
- **动作注册**：支持自定义动作通过装饰器注册
- **技能集成**：动态加载和注册外部技能

### **2. 异步优先**

- 全面使用 `async/await`
- 支持并发动作执行（`multi_act()`）
- 事件驱动的浏览器控制
- 流式响应（SSE）

### **3. 容错和恢复**

- 完整的异常处理链
- 超时保护（步骤级、会话级）
- 重试机制
- 信号处理（Ctrl+C 暂停/恢复）

### **4. 观测性**

```python
# 三层观测：
1. 日志系统 (logging_config.py)
   - 控制台输出
   - 文件日志
   - 自定义日志级别

2. 遥测系统 (telemetry/)
   - 事件收集
   - 性能指标
   - 错误追踪

3. 可观测性装饰器 (observability.py)
   - lmnr 集成（可选）
   - 调试追踪
   - 性能分析
```

### **5. 多种使用方式**

```python
# 方式 1：编程 API
agent = Agent(task="...", llm=ChatBrowserUse())
await agent.run()

# 方式 2：CLI (交互式)
browser-use

# 方式 3：MCP 服务
BrowserUseServer() # 与 Claude 集成

# 方式 4：沙箱执行
@sandbox
async def my_task(browser: Browser):
    ...
```

## 📊 性能优化

| 优化点 | 实现 | 效果 |
|--------|------|------|
| **Prompt Caching** | LLM 消息缓存 | 降低 LLM 成本 90% |
| **Flash Mode** | 简化的输出格式 | 加快响应速度 |
| **Memory Compaction** | 历史压缩 | 减少上下文长度 |
| **Parallel Actions** | 多动作并发 | 提高执行效率 |
| **Element Dedup** | DOM 元素去重 | 减少噪声 |
| **Vision Optimization** | 智能截图 | 降低 LLM 负担 |
