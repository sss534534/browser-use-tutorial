# 4. Agent 设计模式深入分析

## 🏗️ Agent 的核心设计模式

browser-use 的 Agent 采用多个经典设计模式的组合，形成一个高效的自主代理系统。

### **1. 分层执行架构（Layered Execution Architecture）**

```
Agent.run() [最高层 - 会话管理]
    ├─ Signal Handler (CTRL+C 暂停/恢复)
    ├─ Event Bus (CreateAgentSessionEvent)
    └─ while loop (max_steps)
        │
        └─ Agent.step() [中层 - 步骤执行]
            ├─ Phase 1: _prepare_context()
            ├─ Phase 2: _get_next_action() + _execute_actions()
            ├─ Phase 3: _post_process()
            └─ Phase 4: _finalize()
                │
                └─ Tools.execute() [底层 - 动作执行]
```

**关键特点**：
- 🎯 **三层清晰分离**：会话→步骤→动作
- 🔄 **异常统一处理**：所有异常在 `_handle_step_error` 中处理
- ✅ **finally 块保证清理**：无论成功/失败都执行 `_finalize`

## 🔄 Agent.step() 的五阶段模型

```python
async def step(self) -> None:
    """Execute one step of the task"""
    self.step_start_time = time.time()
    
    try:
        # Phase 0: 特殊处理 - 验证码等待
        captcha_wait = await self.browser_session.wait_if_captcha_solving()
        if captcha_wait and captcha_wait.waited:
            # 重置计时（验证码时间不计入步长）
            self.step_start_time = time.time()
        
        # Phase 1: 准备上下文 (获取浏览器状态)
        browser_state_summary = await self._prepare_context(step_info)
        
        # 清空前一步的状态
        self.state.last_model_output = None
        self.state.last_result = None
        
        # Phase 2: 获取 LLM 决策 + 执行动作
        await self._get_next_action(browser_state_summary)
        await self._execute_actions()
        
        # Phase 3: 后处理
        await self._post_process()
    
    except Exception as e:
        # Phase 4: 错误处理 (统一处理所有异常)
        await self._handle_step_error(e)
    
    finally:
        # Phase 5: 清理 (无论成功/失败都执行)
        await self._finalize(browser_state_summary)
```

### **Phase 1: _prepare_context() - 状态获取**

```python
async def _prepare_context(self, step_info: AgentStepInfo | None = None) -> BrowserStateSummary:
    """准备步骤的上下文：浏览器状态、可用动作、页面分析"""
    
    # 1️⃣ 获取浏览器状态 (DOM、URL、标题、标签页)
    browser_state = await self.browser_session.get_state()
    
    # 2️⃣ 如果需要，提取元素并生成元素索引
    dom_state = await self.dom_service.extract(browser_state)
    
    # 3️⃣ 检测可交互元素（用于 LLM 引用）
    clickable_elements = dom_state.get_clickable_elements()
    
    # 4️⃣ 生成可用动作列表 (click, type, scroll, 自定义动作等)
    available_actions = self.tools.get_available_actions()
    
    # 5️⃣ 构建状态总结给 LLM
    browser_state_summary = BrowserStateSummary(
        dom_state=dom_state,
        url=browser_state.url,
        title=browser_state.title,
        tabs=browser_state.tabs,
        screenshot=screenshot_base64,  # 如果 use_vision=True
    )
    
    return browser_state_summary
```

**设计特点**：
- 📊 **完整快照**：每步都获取完整的浏览器状态
- 🎬 **可视化支持**：可选截图用于 LLM 视觉理解
- 🔍 **元素索引**：DOM 元素按顺序编号，LLM 可直接引用
- 📝 **无状态**：不依赖前一步的状态

### **Phase 2: _get_next_action() - LLM 决策 + 动作执行**

```python
async def _get_next_action(self, browser_state_summary: BrowserStateSummary) -> None:
    """调用 LLM 获取下一步动作"""
    
    # 1️⃣ 构建 LLM 消息
    messages = self.message_manager.create_messages(
        current_state=browser_state_summary,
        history=self.state.history[-N:],  # 压缩的历史
        previous_action_result=self.state.last_result,  # 前一步结果
        memory=self.message_manager.compacted_memory,  # 长期记忆
    )
    
    # 2️⃣ 调用 LLM (带超时和重试)
    try:
        llm_response = await asyncio.wait_for(
            self.llm.chat(messages),
            timeout=self.settings.llm_timeout
        )
    except asyncio.TimeoutError:
        raise AgentError(f"LLM timeout after {self.settings.llm_timeout}s")
    
    # 3️⃣ 解析 LLM 输出
    output = AgentOutput.parse(llm_response)
    
    # 4️⃣ 验证动作有效性
    for action in output.action:
        self.tools.validate_action(action)
    
    # 5️⃣ 保存状态供后续阶段使用
    self.state.last_model_output = output
    
    # 日志记录
    self.logger.info(f"💭 Next goal: {output.next_goal}")
    for action in output.action:
        self.logger.info(f"  → {action.name}({action.args})")

async def _execute_actions(self) -> None:
    """执行 LLM 输出的所有动作"""
    if self.state.last_model_output is None:
        raise ValueError('No model output to execute')
    
    # multi_act 有两层保护防止对陈旧 DOM 的动作执行
    result = await self.multi_act(self.state.last_model_output.action)
    self.state.last_result = result
```

**关键特性**：

| 特性 | 实现 | 优点 |
|------|------|------|
| **LLM 超时** | `asyncio.wait_for(..., timeout=90s)` | 防止无限等待 |
| **动作验证** | `validate_action()` 在执行前 | 早期发现错误 |
| **批处理** | `multi_act(list[ActionModel])` | 支持并发执行 |
| **DOM 保护** | 执行前后检查 URL 和焦点 | 防止对陈旧 DOM 的操作 |

### **Phase 3: _post_process() - 后处理**

```python
async def _post_process(self) -> None:
    """后处理：评估、记忆更新、完成检查"""
    
    # 1️⃣ 检查任务是否完成
    if any(result.is_done for result in self.state.last_result):
        self.state.task_complete = True
        self.logger.info('✅ Task marked as done by action result')
        return
    
    # 2️⃣ 更新记忆
    if self.state.last_model_output.memory:
        self.message_manager.update_memory(
            self.state.last_model_output.memory
        )
    
    # 3️⃣ 检查失败计数（连续失败太多次）
    if self._is_step_failed():
        self.state.consecutive_failures += 1
        if self.state.consecutive_failures >= self.settings.max_failures:
            raise InterruptedError('Max failures reached')
    else:
        self.state.consecutive_failures = 0  # 重置
    
    # 4️⃣ 检测循环（重复的动作）
    loop_nudge = self.state.loop_detector.get_nudge_message()
    if loop_nudge:
        self.logger.warning(f'⚠️ {loop_nudge}')
        # 注入到下一步的消息中
        self.message_manager.inject_nudge(loop_nudge)
    
    # 5️⃣ 检查是否需要重新规划
    if not self.state.plan or self._should_replan():
        self.logger.info('📋 Generating new plan...')
        self.state.plan = await self._generate_plan()
```

**自适应机制**：
- 🔄 **失败重试**：连续失败时自动调整策略
- 🔁 **循环检测**：检测重复动作并提醒 LLM
- 📊 **自动规划**：步数过多或失败时重新生成计划
- 💾 **记忆管理**：压缩历史防止上下文爆炸

### **Phase 4: _handle_step_error() - 错误处理**

```python
async def _handle_step_error(self, e: Exception) -> None:
    """统一的错误处理"""
    
    self.logger.error(f'❌ Step error: {e}', exc_info=True)
    
    # 区分错误类型，采取不同策略
    if isinstance(e, InterruptedError):
        # 用户暂停、停止或达到最大步数
        self.state.stopped = True
        raise
    
    elif isinstance(e, TimeoutError):
        # 超时（LLM 或步长超时）
        self.state.consecutive_failures += 1
        error_msg = f'Timeout: {str(e)}'
        
    elif isinstance(e, ValidationError):
        # LLM 输出验证失败
        self.state.consecutive_failures += 1
        error_msg = f'Invalid output: {str(e)}'
    
    elif isinstance(e, AgentError):
        # 代理特定错误
        self.state.consecutive_failures += 1
        error_msg = str(e)
    
    else:
        # 未知错误，也要计数
        self.state.consecutive_failures += 1
        error_msg = f'Unexpected error: {str(e)}'
    
    # 记录到历史（即使失败也要记录）
    self.history.add(AgentHistory(
        error=error_msg,
        step_number=self.state.n_steps,
    ))
    
    # 如果启用了"失败后最后一次响应"，不抛出
    if (self.state.consecutive_failures < self.settings.max_failures + 
        int(self.settings.final_response_after_failure)):
        return  # 继续下一步
    else:
        raise InterruptedError(f'Max failures reached: {error_msg}')
```

**错误恢复策略**：
```
首次失败 → 继续 → 再次失败 → 计数+1 → 达到 max_failures
                                        ↓
                        如果 final_response_after_failure=True:
                        尝试最后一次恢复性调用 LLM
                        ↓
                        否则中止
```

### **Phase 5: _finalize() - 清理**

```python
async def _finalize(self, browser_state_summary: BrowserStateSummary | None) -> None:
    """最终化：记录步骤、更新计数"""
    
    # 1️⃣ 记录步骤到历史
    step_duration = time.time() - self.step_start_time
    metadata = StepMetadata(
        duration_seconds=step_duration,
        model_used=self.llm.model_name,
        tokens_used=self.token_cost_service.get_step_tokens(),
    )
    
    self.history.add(AgentHistory(
        step_number=self.state.n_steps,
        state=browser_state_summary,
        model_output=self.state.last_model_output,
        result=self.state.last_result,
        metadata=metadata,
    ))
    
    # 2️⃣ 递增步数计数器
    self.state.n_steps += 1
    
    # 3️⃣ 检查是否需要压缩内存
    if len(self.history.history) > self.settings.max_history_items:
        await self.message_manager.compact_memory()
    
    # 4️⃣ 发送遥测事件
    self.eventbus.dispatch(UpdateAgentStepEvent.from_agent(self))
    
    # 5️⃣ 主动垃圾回收（防止内存泄漏）
    if self.state.n_steps % 10 == 0:  # 每 10 步一次
        gc.collect()
    
    # 6️⃣ 暂停检查
    if self.state.paused:
        self.logger.info('⏸️  Agent paused, waiting to resume...')
        await self._external_pause_event.wait()
```

## 🎯 Agent.run() 的会话管理

```python
async def run(
    self, 
    max_steps: int = 500,
    on_step_start: AgentHookFunc | None = None,
    on_step_end: AgentHookFunc | None = None,
) -> AgentHistoryList:
    """最高层：会话管理"""
    
    # 1️⃣ 设置信号处理 (CTRL+C)
    signal_handler = SignalHandler(
        loop=asyncio.get_event_loop(),
        pause_callback=self.pause,       # 第一次 CTRL+C: 暂停
        resume_callback=self.resume,     # 继续
        exit_on_second_int=True,         # 第二次 CTRL+C: 退出
    )
    signal_handler.register()
    
    try:
        # 2️⃣ 初始化会话
        await self.browser_session.start()
        
        # 3️⃣ 发送事件 (CreateAgentSessionEvent)
        self.eventbus.dispatch(CreateAgentSessionEvent.from_agent(self))
        
        # 4️⃣ 执行初始动作 (如果提供)
        if self.initial_actions:
            await self._execute_initial_actions()
        
        # 5️⃣ 主循环
        while self.state.n_steps <= max_steps:
            # 检查暂停/停止
            if self.state.paused:
                await self._external_pause_event.wait()
            
            if self.state.stopped:
                raise InterruptedError('Stopped by external request')
            
            # 执行一步
            await self._execute_step(
                step=self.state.n_steps - 1,
                max_steps=max_steps,
                step_info=step_info,
                on_step_start=on_step_start,
                on_step_end=on_step_end,
            )
            
            # 检查完成条件
            if self.state.task_complete:
                self.logger.info('✅ Task completed!')
                break
        
        # 6️⃣ 返回历史
        return self.history
    
    except KeyboardInterrupt:
        self.logger.info('Interrupted by user')
        return self.history
    
    finally:
        # 7️⃣ 清理
        signal_handler.unregister()
        await self.eventbus.stop()
        await self.browser_session.close()
```

## 🔀 设计模式分类

### **模式 1：模板方法模式 (Template Method)**

```
Agent.step() 定义了固定的步骤流程：
├─ Phase 1: _prepare_context()
├─ Phase 2: _get_next_action() + _execute_actions()
├─ Phase 3: _post_process()
├─ Exception Handler: _handle_step_error()
└─ Finalization: _finalize()

具体实现可被子类覆盖（不同的 Agent 类型）
```

**优势**：
- 📋 算法结构清晰
- 🔄 易于添加新的处理阶段
- 🎯 保证步骤的一致性

### **模式 2：责任链模式 (Chain of Responsibility)**

```python

    await self._prepare_context()      # 可能抛异常
    await self._get_next_action()      # 可能抛异常
```

## 🔬 关键算法实现

### **1. Loop Detection（循环��测）**

```python
class ActionLoopDetector(BaseModel):
    """检测 Agent 是否陷入循环（重复相同动作）"""
    
    window: list[str] = Field(default_factory=list)  # 最近 20 个 DOM 状态
    max_repetition_count: int = 0
    
    def record_page_state(self, url: str, dom_text: str, element_count: int):
        """记录当前页面状态的指纹"""
        fingerprint = f"{url}|{hash(dom_text)}|{element_count}"
        self.window.append(fingerprint)
        self.window = self.window[-20:]  # 保留最近 20 个
        
        # 更新重复计数
        self._update_repetition_stats()
    
    def get_nudge_message(self) -> str | None:
        """返回逐步升级的警告"""
        if self.max_repetition_count < 3:
            return None  # 还好
        elif self.max_repetition_count == 3:
            return "⚠️ Agent may be in a loop. Consider clicking different elements or scrolling."
        elif self.max_repetition_count == 5:
            return "🔁 Agent detected in a loop. Try a different approach or use go_back."
        else:
            return "🔴 Agent strongly appears stuck in a loop. Consider stopping or adjusting task."
```

**工作流**：
```
每步执行后：
1. 记录页面指纹 (URL + DOM hash + 元素数)
2. 在 20 步的滑动窗口中计算重复
3. 如果相同状态出现 3 次+ → 提醒 LLM
4. 逐步升级警告强度
```

### **2. Memory Compaction（内存压缩）**

```python
async def compact_memory(self) -> None:
    """将历史步骤压缩为摘要"""
    
    # 获取前 N 步的历史
    history_to_compact = self.history[:20]
    
    # 构建压缩提示
    compact_prompt = f"""
    Summarize the following steps into a concise bullet-point memory:
    {format_history(history_to_compact)}
    
    Include only the most important information:
    - URLs visited
    - Important findings
    - Failed attempts
    """
    
    # 调用 LLM 生成摘要
    summary = await self.page_extraction_llm.chat([
        SystemMessage(compact_prompt),
    ])
    
    # 更新状态
    self.compacted_memory += f"\n{summary}"
    self.history = self.history[20:]  # 移除已压缩的历史
    self.compaction_count += 1
```

**Token 成本优化**：
```
无压缩：每步 + 前 N 步的完整历史 = O(n)
有压缩：每步 + 摘要 + 最近 5 步 = O(1) + O(5) = O(1)

示例：
100 步任务
无压缩：步骤 1-100 都包含完整历史 → 总 token = 1+2+3+...+100 = 5050
有压缩：
- 步骤 1-20: 完整
- 步骤 21-40: 摘要 + 最近 5 步
- 步骤 41-60: 摘要 + 最近 5 步
- ...
→ 总 token ≈ 20 + 25*4 = 120

节省：5050 / 120 ≈ 42x
```

### **3. Multi-Act Execution（多动作执行）**

```python
async def multi_act(self, actions: list[ActionModel]) -> list[ActionResult]:
    """执行多个动作，但有两层 DOM 保护"""
    
    results: list[ActionResult] = []
    
    for i, action in enumerate(actions):
        # 第 1 层: 静态检查
        if action.terminates_sequence:  # navigate, search, go_back, switch
            # 其他队列中的动作会被中止
            results.append(await self.tools.execute(action))
            break
        
        # 第 2 层: 运行时检查
        pre_action_url = self.browser_session.url
        pre_action_target = self.browser_session.get_focused_target()
        
        result = await self.tools.execute(action)
        results.append(result)
        
        # 执行后检查：URL 或焦点是否改变?
        post_action_url = self.browser_session.url
        post_action_target = self.browser_session.get_focused_target()
        
        if pre_action_url != post_action_url or pre_action_target != post_action_target:
            # 页面已改变，停止执行剩余动作
            self.logger.warning(
                f'Page changed after action {i}. '
                f'Aborting remaining {len(actions) - i - 1} actions.'
            )
            break
    
    return results
```

**两层保护的好处**：
```
情景 1: navigate 动作在队列中
├─ 静态检查：识别出 navigate 是 terminates_sequence=True
└─ 立即中止其他动作（它们对新页面是过时的）

情景 2: 点击触发了导航（如链接）
├─ 执行 click
├─ 运行时检查：URL 改变了！
└─ 立即中止剩余动作
```

## 🎯 总结：Agent 的三大核心模式

| 模式 | 应用 | 效果 |
|------|------|------|
| **分层执行** | run() → step() → execute() | 清晰的抽象层次 |
| **模板方法** | Phase 1-5 的固定流程 | 一致的步骤结构 |
| **观察者** | EventBus 事件驱动 | 解耦的反馈机制 |
| **状态机** | AgentState 管理 | 可预测的状态转换 |
| **策略** | LLM 可互换 | 灵活的模型选择 |
| **装饰器** | @observe/@time_execution | 自动的可观测性 |
| **责任链** | 统一异常处理 | 强大的错误恢复 |

这些模式的组合使得 Agent 既**强大和灵活**，又**易于理解和维护**。
